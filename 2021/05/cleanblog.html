Clean blog theme 설정

Default로 설정된 minima theme을 jekyll-theme-clean-blog로 변경 : Start Bootstrap site

Powershell로 메모장으로 Gemfile 열기

PS E:\Github\blog\myblog.github.io> notepad .\Gemfile

Gemfile을 아래와 같이 편집 (minima를 #로 주석처리 후 clean-blog를 추가해 줌)

# This is the default theme for new Jekyll sites. You may change this to anything you like.
#gem "minima", "~> 2.5"
gem "jekyll-theme-clean-blog"

편집 후 bundle install을 호출하여 dependency가 생긴 theme을 다운로드, 설치 하도록 한다.

Jekyll blog의 _config.yml 파일을 수정하여, 새로 설치한 theme을 default로 설정한다.

PS E:\Github\blog\myblog.github.io> notepad .\_config.yml

# Build settings
#theme: minima
theme: jekyll-theme-clean-blog
plugins:
  - jekyll-feed

Blog를 local에서 실행 해 봄

bundle exec를 수행 하여 jekyll 웹 서버를 기동 시킨다. 아무런 에러가 없다면 제대로 설치 된 것이다. 웹 브라우져로 Server address 의 url을 open하게 되면 생성된 blog에 접속됨을 확인 할 수 있다.

index.html, about.html, contact.html, posts/index.html 파일을 생성

PS E:\Github\blog\myblog.github.io> notepad .\index.html
PS E:\Github\blog\myblog.github.io> notepad .\about.html
PS E:\Github\blog\myblog.github.io> notepad .\contact.html
PS E:\Github\blog\myblog.github.io> mkdir posts
PS E:\Github\blog\myblog.github.io> notepad .\posts\index.html

index.html 파일 편집

---
layout: home
background: '/PATH_TO_IMAGE'
---

나머지 파일 편집 (about.html, contact.html, posts/index.html)

---
layout: page
title: Page Title
description: This is the page description.
background: '/PATH_TO_IMAGE'
---

_posts 폴더내에 실제로 내가 올릴 글 (markdown file로 md 확장자)을 날짜를 포함해서 만든다.

---
layout: post
title: "Post Title"
subtitle: "This is the post subtitle."
date: YYYY-MM-DD HH:MM:SS
background: '/PATH_TO_IMAGE'
---

Blog 정보를 변경

내 Blog 이름 및 Description을 입력해 준다. 연락받을 email및 sns도 변경

title: Your awesome title
email: your-email@example.com
description: >- # this means to ignore newlines until "baseurl:"
  Write an awesome description for your new site here. You can edit this
  line in _config.yml. It will appear in your document head meta (for
  Google search results) and in your feed.xml site description.
baseurl: "" # the subpath of your site, e.g. /blog
url: "" # the base hostname & protocol for your site, e.g. http://example.com
twitter_username: jekyllrb
github_username:  jekyll

blog를 다시 성성 및 기동 시킨다.

PS E:\Github\blog\myblog.github.io> bundle exec jekyll serve

Pagenate v2 설정

Gemfile 수정하여 jekyll-pagenate-v2 설치

PS E:\Github\blog\myblog.github.io> notepad .\Gemfile

group :jekyll_plugins do
  gem "jekyll-feed", "~> 0.12"
  gem "jekyll-paginate-v2"
end

PS E:\Github\blog\myblog.github.io> bundle install

_config 설정

plugin에 paginate-v2를 추가하고, pagination 항목을 넣어준다.

plugins:
  - jekyll-feed
  - jekyll-paginate-v2

permalink: /:year/:month/
pagination:
  enabled: true
  per_page: 12
  sort_reverse: true
  sort_field: 'date'
  title: ':title'
  trail:
    before: 2
    after: 2

index.html 파일에 설정

---
layout: home
pagination: 
  enabled: true
---

posts/index.html 파일 설정

---
layout: page
title: Page Title
description: This is the page description.
pagination: 
  enabled: true
  category: test
---

다시 실행 시킨다.

PS E:\Github\blog\myblog.github.io> bundle exec jekyll serve

github.io에 배포하기

_site에 생성된 파일을 github folder로 copy한다.
